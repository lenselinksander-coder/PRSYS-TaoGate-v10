Goed.
Hier is een preciese, uitvoerbare prompt die je in Replit Agent kunt plakken om de response-shape te harmoniseren (zodat Clinical en EU dezelfde structuur hebben en de UI nooit meer undefined krijgt).

⸻

🔧 PROMPT VOOR AGENT

You are modifying the backend API so that all /api/classify responses share one unified response shape, regardless of whether the result comes from:
	•	ClinicalGate (hard stop)
	•	EU scope classification

Goal

Unify the response model so frontend can always rely on:

{
  status: string;
  olympia: string | null;
  layer: string;
  pressure: string;
  escalation: string | null;
  reason: string | null;
  winningRule: any | null;
  signals: any | null;
}

No conditional frontend branching required.

⸻

Step 1 — Modify ClinicalGate return block in routes.ts

Find this block:

if (gate.status === "BLOCK" || gate.status === "ESCALATE_HUMAN") {
  return res.json({
    status: gate.status,
    layer: gate.layer,
    band: gate.band,
    pressure: gate.pressure,
    escalation: gate.escalation,
    reason: gate.reason,
    signals: gate.signals,
  });
}

Replace it with:

if (gate.status === "BLOCK" || gate.status === "ESCALATE_HUMAN") {
  return res.json({
    status: gate.status,
    olympia: gate.band,          // mapped for UI compatibility
    layer: gate.layer,           // always "CLINICAL"
    pressure: gate.pressure,
    escalation: gate.escalation,
    reason: gate.reason,
    winningRule: null,           // unify shape
    signals: gate.signals,
  });
}


⸻

Step 2 — Modify normal EU/scope return

Ensure your final classification return (after classifyWithScope) matches the SAME shape:

Replace any existing return block with:

return res.json({
  status: classification.status,
  olympia: classification.olympia ?? null,
  layer: classification.layer,
  pressure: classification.pressure,
  escalation: classification.escalation ?? null,
  reason: classification.reason ?? null,
  winningRule: classification.winningRule ?? null,
  signals: null,
});


⸻

Step 3 — Remove legacy fields

Ensure:
	•	No band is returned directly
	•	No raw winningRule.pressure is exposed outside unified structure
	•	No undefined values (use ?? null)

⸻

After change, test:

	2.	Observatie

No undefined values in UI.

⸻

Constraints
	•	Do NOT change ClinicalGate logic.
	•	Do NOT change classifyWithScope logic.
	•	Only harmonize response shape.
	•	Ensure TypeScript compiles cleanly.
	•	Remove orphaned code if present.

⸻

End of modification.
